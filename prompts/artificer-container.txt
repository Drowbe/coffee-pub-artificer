===============================
JSON TEMPLATE FOR CONTAINERS
===============================

Containers use type "loot" and flags.artificer.type "container". Include core Artificer fields for recipe matching.

CONTAINER FAMILIES (family):
- Glass (Beaker, Vial)
- Ceramic (Mortar and Pestle)
- Metal (Crucible)
- General (Mixing Bowl)
- Cloth (Herb Bag)

TAGS for recipe matching (primaryTag, secondaryTags):
- Beaker: Liquid, Heat-Safe
- Vial: Liquid, Small
- Mortar and Pestle: Grinding, Dry
- Crucible: Metal, High-Heat
- Mixing Bowl: General, Mixing
- Herb Bag: Herbal, Infusion

{
  "name": "CONTAINERNAME",
  "type": "loot",

  "description": "DESCRIPTION",
  "weight": WEIGHT,
  "price": PRICE,
  "rarity": "common",
  "system": {
    "source": { "value": "Artificer", "custom": "Artificer", "license": "OGL" }
  },
  "flags": {
    "coffee-pub-artificer": {
      "type": "container",
      "family": "FAMILY",
      "primaryTag": "PRIMARYTAG",
      "secondaryTags": ["TAG1", "TAG2"],
      "tier": 1,
      "rarity": "Common",
      "quirk": null,
      "biomes": []
    }
  }
}

REQUIRED:
- source: Always set. Use system.source with value, custom, and license.
- license: Use "OGL", "CC-BY-4.0", or project license. Never leave empty.
- family: One of Glass, Ceramic, Metal, General, Cloth.
- primaryTag, secondaryTags: For recipe/container matching. Tags help recipes require specific containers (e.g. potion recipes â†’ Liquid container).

Note: The Artificer import currently requires type "ingredient", "component", or "essence". Container support may need to be added to the import utility. Until then, create these items manually via the Create Item form with flags.artificer.type = "container".
